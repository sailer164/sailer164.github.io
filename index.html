Java 服务端技术选型

移动互联网的出现导致之前的web mvc模式发生了变化
1、很多的移动app不需要服务端的页面，仅仅需要能够提供http数据请求的服务
2、需要快速搭建的架构，之前的由tomcat 加载web.xml等一系列配置已显得繁冗复杂

这就需要一个具有以下几个特点的web架构：
1、纯后台的，不需要任何页面
2、简单，尽量不涉及复杂的web配置，如之前的web.xml
3、支持高并发
4、易上手，易推广

这种架构毫无疑问的就想到了restful，代表性状态传输（Representational State Transfer，REST）在 Web 领域已经得到了广泛的接受，是基于 SOAP 和 Web 服务描述语言（Web Services Description Language，WSDL）的 Web 服务的更为简单的替代方法。 接口设计方面这一转变的关键证据是主流 Web 2.0 服务提供者（包括 Yahoo、Google 和 Facebook）对 REST 的采用，这些提供者弃用或放弃了基于 SOAP 和 WSDL 的接口，而采用了更易于使用、面向资源的模型来公开其服务，并且已经纳入到JDK1.6当中了

restful几个特点：
1、REST 要求开发人员显式地使用 HTTP 方法，并且使用方式与协议定义一致。 这个基本 REST 设计原则建立了创建、读取、更新和删除（create, read, update, and delete，CRUD）操作与 HTTP 方法之间的一对一映射。 根据此映射：
若要在服务器上创建资源，应该使用 POST 方法。
若要检索某个资源，应该使用 GET 方法。
若要更改资源状态或对其进行更新，应该使用 PUT 方法。
若要删除某个资源，应该使用 DELETE 方法。

2、REST Web 服务需要扩展以满足日益提高的性能要求。 具有负载平衡和故障转移功能、代理和网关的服务器集群通常以形成服务拓扑的方式进行组织，从而允许根据需要将请求从一个服务器路由到另一个服务器，以减少 Web 服务调用的总体响应时间。 要使用中间服务器扩大规模，REST Web 服务需要发送完整、独立的请求；也就是说，发送的请求包括所有需要满足的数据，以便中间服务器中的组件能够进行转发、路由和负载平衡，而不需要在请求之间在本地保存任何状态。
3、REST Web 服务 URI 的直观性应该达到很容易猜测的程度。 将 URI 看作是自身配备文档说明的接口，开发人员只需很少（如果有的话）的解释或参考资料即可了解它指向什么，并获得相关的资源。 为此，URI 的结构应该简单、可预测且易于理解。
4、传输数据的形式可以为xml格式也可以是json格式

确定好rest作为后台入口后，紧接着就是rest技术选型，看了spring和cxf的样例代码都比较简单，cxf更接近原生态，有PATH、GET等JAX-RS的注解，客户端需自己封装下HTTPCLIENT的返回数据类型，而spring已经提供了更为封装的RestController注解，并且客户端已经封装在RestTemplate类中，再者spring rest与spring框架天然的结合优势最终确定了spring的rest实现，spring rest 的maven配置为

org.springframework
spring-web
${springframework.version}

确定好rest实现技术后就要确定web容器，选用tomcat还是jetty，spring boot starter默认集成的是嵌入式的tomcat，看了下二者的区别：
1、jetty 架构更为简单
2、 jetty对新的规范如google提出的新型网络协议SPDY支持较为快速
3、 jetty体积小，更容易部署
4、 jetty易扩展，源于其handler的架构设计

二者的性能差异其实并不大，由于后续主要面临的是移动互联网应用，考虑其部署和易扩展还是选定了jetty，在查阅资料的同时发现google也是选用的jetty，maven配置如下：

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter</artifactId>
        <version>1.1.3.RELEASE</version>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
        <version>1.1.3.RELEASE</version>
        <exclusions>
            <exclusion>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter-tomcat</artifactId>
            </exclusion>
        </exclusions>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jetty</artifactId>
        <version>1.1.3.RELEASE</version>
    </dependency>

数据库毫无疑问选用了mysql，dao这块就选用了简单的mybatis，连接池选用了号称最快的druid（阿里开源的），整个web的后台架构就这么基本确定了
